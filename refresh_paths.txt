# ============================================================
# 定时主动刷新目标 - refresh_paths.txt
# ============================================================
#
# 【填什么？】
# ⚠️ 核心注意：不要填网盘的“真实路径”，要填 STRM 引擎映射出来的“虚拟路径”！
#
# 举个例子：
# 你的网盘真实路径是：/天翼云/家庭/电影
# 你的STRM引擎挂载在：/strm
# OpenList 会把真实路径的最后一级，映射为引擎下的虚拟目录，即：/strm/电影
# 那么，你在这里应该填：/strm/电影 （或者想刷新所有，就直接填 /strm）
#
# 【为什么填这个？】
# OpenList 的 STRM 引擎有时需要有人去“点”一下目录，才会触发文件的生成。
# 这里填写的路径，程序会每隔几分钟去“模拟点击”（调用 API 获取列表），
# 从而唤醒 OpenList，强制它生成或更新最新的 strm 文件到 A 区。
#
# 【智能交叉逻辑】
# 1. 深度清理：如果你填的路径（如 /测试a/动漫），刚好属于 strm_engine_paths.txt 的管辖范围，
#    程序刷新完后，会顺手对比本地，把云端已经不存在的废弃 strm 文件删掉（冗余清理）。
# 2. 只读保护：如果你填错了，填成了网盘真实路径（如 /天翼云/电影），程序发现它不在引擎白名单里，
#    为了防灾，程序只会对它执行“只读刷新”，绝对不会去清理本地文件，防止把你的媒体库清空！
#
# 示例：
# /strm/电影
# /strm/番剧
# /测试a/特定动漫
#
# ============================================================
/测试a
